# Emoji Copy Site MVP Tasks

## 目標
emojicopy.com のMVP（Minimum Viable Product）を作成する

## 共通ルール
- 各実装タスクに対してテストコードを書く
- Bunのtest機能を使用 (`bun:test`)
- テストファイルは `__tests__/` ディレクトリまたは `*.test.ts` ファイル

---

## Phase 1: 基盤設定

- [ ] **Task 1: 絵文字データの準備**
  - カテゴリ別絵文字データを作成
  - JSONファイルまたはTypeScriptモジュールとして定義
  - カテゴリ: Smileys, People, Animals, Food, Activities, Travel, Objects, Symbols, Flags
  - **テスト**: データが正しく読み込まれること、各カテゴリが絵文字を含んでいること

- [ ] **Task 2: テスト環境のセットアップ**
  - `bun test` コマンドが動作するように設定
  - テスト用のスクリプトを package.json に追加
  - **テスト**: サンプルテストが実行できること

---

## Phase 2: UIコンポーネント

- [ ] **Task 3: 絵文字グリッドコンポーネント**
  - 絵文字をグリッド表示するコンポーネントを作成
  - レスポンシブデザイン（モバイルフレンドリー）
  - **テスト**: コンポーネントが正しくレンダリングされること、絵文字が表示されること

- [ ] **Task 4: カテゴリナビゲーション**
  - カテゴリを選択するタブまたはサイドバーを作成
  - カテゴリ切り替え時に絵文字リストを更新
  - **テスト**: カテゴリクリックで正しい絵文字が表示されること

- [ ] **Task 5: 検索バー**
  - 絵文字をキーワード検索する機能
  - 検索結果をリアルタイムに表示
  - **テスト**: 検索キーワードに一致する絵文字が表示されること

---

## Phase 3: コア機能

- [ ] **Task 6: クリップボードコピー機能**
  - 絵文字クリックでクリップボードにコピー
  - コピー成功のフィードバック（トーストまたは視覚的フィードバック）
  - **テスト**: クリック後にクリップボードに絵文字がコピーされること

- [ ] **Task 7: 最近使った絵文字（履歴）**
  - コピーした絵文字の履歴を保存（localStorage使用）
  - 履歴を表示するセクション
  - **テスト**: コピーした絵文字が履歴に保存されること、ページリロード後も保持されること

---

## Phase 4: スタイリングとアクセシビリティ

- [ ] **Task 8: ダークモード対応**
  - 既存のダークモード変数を活用
  - 絵文字が両方のモードで見やすいことを確認
  - **テスト**: ライトモード・ダークモード両方で正しく表示されること

- [ ] **Task 9: ホバー効果とアクションフィードバック**
  - 絵文字のホバー時の視覚的フィードバック
  - コピー時のアニメーション
  - **テスト**: ホバー状態が正しく適用されること

---

## Phase 5: パフォーマンスと最適化

- [ ] **Task 10: 大量絵文字の仮想化または遅延読み込み**
  - パフォーマンスのための表示最適化
  - **テスト**: 大量データでも画面が固まらないこと

---

## 実装順序の推奨
1. Task 1-2: 基盤準備
2. Task 3: 基本的なグリッド表示
3. Task 6: コピーフィードバック
4. Task 4: カテゴリ機能
5. Task 5: 検索機能
6. Task 7-9: UX改善
7. Task 10: パフォーマンス最適化（必要に応じて）

## 技術スタック
- Next.js 16 (App Router)
- React 19
- TypeScript
- Tailwind CSS v4
- Bun (パッケージマネージャー・テストランナー)
